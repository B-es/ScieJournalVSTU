# Спецификация дизайна (DS)

---

## 1. Принципы дизайна

1. Сайт для работы с научными статьями, не для демонстрации визуальных эффектов.
2. Минимум акцентных цветов — один основной акцент (`color.primary`), остальные цвета служебные (статусы, текст, фон).
3. Разделение страниц на 2 типа: информационные и личный кабинет.
4. Информационные страницы во всю сетку страницы. Страницы личного кабинета делят сетку на 2 блока по горизонтали: левый блок меню — 2 колонки, правый блок информации — 10 колонок.
5. Статус статьи всегда обозначается не только цветом, но и текстовой меткой (см. компонент «Статус-бейдж») — на сайте, где решения (принята/отклонена) критичны для пользователя, полагаться только на цвет недопустимо.

**Референсы / вдохновение:** https://www.mdpi.com/, https://link.springer.com, https://www.researchgate.net/topic/Computer-Science

### Типографика

*Исходные значения `type.h1`/`type.h2` (оба 32px) и `type.body` (24px) не давали визуальной иерархии и были слишком крупными для основного текста на content-heavy сайте — шкала скорректирована ниже, добавлен уровень `h3` для заголовков карточек.*

| Токен | Шрифт | Размер | Начертание | Line-height | Использование |
|---|---|---|---|---|---|
| type.h1 | DM Sans | 32 | Bold | 40 | заголовок страницы |
| type.h2 | DM Sans | 24 | SemiBold | 32 | заголовок секции |
| type.h3 | DM Sans | 20 | SemiBold | 28 | заголовок карточки (статья, выпуск), заголовок формы |
| type.body | DM Sans | 16 | Regular | 24 | основной текст |
| type.body.large | DM Sans | 18 | Regular | 28 | аннотация на странице статьи |
| type.caption | DM Sans | 14 | Regular | 20 | подписи, метаданные, вспомогательный текст |
| type.button | DM Sans | 16 | Medium | 20 | текст кнопок и интерактивных элементов |

### Отступы (Spacing)

| Токен | Значение | Использование |
|---|---|---|
| spacing.xs | 4px | между иконкой и текстом, внутри тегов |
| spacing.sm | 8px | между связанными элементами (лейбл + значение) |
| spacing.md | 16px | внутренние отступы карточек, между полями формы |
| spacing.lg | 24px | между блоками/секциями внутри страницы |
| spacing.xl | 40px | между крупными секциями страницы (например, между блоками главной) |

- Колонок на desktop: 12
- Ширина контейнера: 1340px
- Gutter (промежуток между колонками): 18px
- Личный кабинет: левая колонка меню — 2/12, правая колонка контента — 10/12, с gutter между ними

### Радиусы

| Токен | Значение | Использование |
|---|---|---|
| radius.sm | 6px | поля ввода, теги, бейджи |

### Тени

*Раздел был помечен «нет», но карточкам и модалкам нужны состояния наведения/подъёма — добавлены значения.*

| shadow.sm | `0 1px 3px rgba(16,24,40,0.08)` | карточки в покое |
| shadow.md | `0 4px 12px rgba(16,24,40,0.12)` | карточки при hover, выпадающие списки |

- Набор иконок: Lucide
- Толщина линии / стиль: outline, stroke-width 1.5px — согласуется с принципом «минимум визуальных эффектов»
- Размеры:

| Токен | Значение | Использование |
|---|---|---|
| icon.size.sm | 16px | иконки внутри полей ввода, тегов, бейджей |

## 3. Макеты

*Файлов макетов нет — структура блоков описана сверху вниз для двух типов страниц (принцип 3–4 раздела 1).*

### Информационная страница (Desktop, 12 колонок)

```
┌─────────────────────────────────────────────────────────────┐
│ Header: логотип · главное меню · переключатель RU/EN · Вход/Регистрация │
├─────────────────────────────────────────────────────────────┤
│ Контентный блок 1 (во всю сетку, 12 колонок)                  │
├─────────────────────────────────────────────────────────────┤
│ Контентный блок 2 (во всю сетку либо сетка карточек 3-4 в ряд) │
├─────────────────────────────────────────────────────────────┤
│                        ...                                    │
├─────────────────────────────────────────────────────────────┤
│ Footer: вспомогательные страницы, копирайт, соцссылки          │
└─────────────────────────────────────────────────────────────┘
```

### Страница личного кабинета (Desktop, 2 + 10 колонок)

```
┌─────────────────────────────────────────────────────────────┐
│ Header (упрощённый): логотип · уведомления · язык · аватар    │
├───────────────┬─────────────────────────────────────────────┤
│ Sidebar (2col)│ Контентная область (10 col)                   │
│ · Пункты меню │  Заголовок раздела (h2)                       │
│   роли        │  Фильтры/действия (например, "Подать статью") │
│ · активный    │  Таблица/список/форма                         │
│   пункт       │                                                │
│   выделен     │                                                │
│   primary     │                                                │
└───────────────┴─────────────────────────────────────────────┘
```

Футер в личном кабинете не показывается — контентная область занимает всё пространство под header.

### Mobile (для обоих типов)

Header сворачивается в компактную строку с бургер-меню. Для информационных страниц контент становится одноколоночным. Для личного кабинета sidebar-меню (2 колонки) превращается в выдвижную панель (drawer) поверх контента, вызываемую бургер-иконкой; контентная область занимает 100% ширины.

## 4. Компоненты

### Компонент: «Выпуск журнала»

| Параметр | Описание |
|---|---|
| Назначение | Карточка журнала, используется на главной странице в блоке «Последний выпуск» и в архиве в блоке «Архив выпусков» |
| Структура | изображение журнала, дата публикации журнала, название, направление, номер, описание журнала, количество статей в журнале, количество страниц в журнале, количество авторов в журнале, язык журнала, кнопки «Посмотреть содержание» и «Открыть выпуск» |
| Размеры | вдоль всех колонок страницы (12/12); в архиве — сетка 2 карточки в ряд на desktop (6/12 каждая) |
| Варианты | «последний выпуск» (крупный, во всю ширину, с акцентной кнопкой «Открыть выпуск» primary) и «архивный выпуск» (компактный, кнопка вторичного вида) |
| Состояния | обычное; hover — `shadow.md` и лёгкое поднятие карточки; выпуск без обложки — плейсхолдер-иконка вместо изображения |
| Доступность | изображение имеет alt-текст с названием и номером выпуска; кнопки доступны с клавиатуры, видимый focus-индикатор; количество статей/страниц продублировано текстом, не только иконкой |

### Компонент: «Компактная карточка статьи»

| Параметр | Описание |
|---|---|
| Назначение | Компактная карточка статьи, используется на главной странице в блоке «Популярные исследования» |
| Структура | тип доступа, количество страниц, вес pdf, дата выпуска, название, авторы, ключевые слова, DOI, кнопка «Приложения» |
| Размеры | занимает 4 колонки страницы (3 карточки в ряд на desktop) |
| Варианты | нет — единый вид для всех статей |

| Доступность | тип доступа (Open Access) обозначается иконкой + текстом (не только цветом/значком замка); DOI выводится как копируемая ссылка с доступным названием для скринридера |

### Компонент: «Статья»

| Параметр | Описание |
|---|---|
| Назначение | Карточка статьи, используется на странице найденных статей в блоке «Найденные статьи» |
| Структура | тип доступа, количество страниц, вес pdf, дата выпуска, название, авторы, аннотация, ключевые слова, DOI, кнопка «Приложения» |
| Размеры | вдоль всех колонок страницы (12/12), список карточек друг под другом |
| Варианты | нет |

| Доступность | структура соответствует семантике списка (`<article>`/`role="article"` в разметке), заголовок статьи — ссылка на страницу статьи, доступная по Tab |

### Компонент: «Статус-бейдж»

| Параметр | Описание |
|---|---|
| Назначение | Отображение текущего статуса статьи во всех списках и на карточке статьи в личном кабинете (На рассмотрении / Требуется доработка / Рецензируется / Принята к публикации / Опубликована / Отклонена) |
| Структура | цветной фон (`color.status.*`), текстовая метка статуса, опционально — иконка (например, часы для «На рассмотрении», галочка для «Опубликована») |
| Размеры | компактный inline-элемент, высота 24px, отступы `spacing.xs`/`spacing.sm` |
| Варианты | 6 вариантов — по числу статусов из раздела 5 PRD |
| Состояния | статичный компонент, без интерактивных состояний |
| Доступность | статус передаётся текстом, а не только цветом (принцип 5 раздела 1); контраст текста к фону ≥ 4.5:1 |

### Компонент: «Кнопка»

| Параметр | Описание |
|---|---|
| Назначение | Основной интерактивный элемент для действий (отправить форму, открыть выпуск, принять приглашение и т.д.) |
| Структура | текстовая метка (`type.button`), опционально иконка слева/справа |
| Размеры | height 40px (обычная), height 32px (компактная, для таблиц) |
| Варианты | primary (заливка `color.primary`, текст `color.text.inverse`) — основное действие на экране; secondary (обводка `color.border`, текст `color.text.primary`) — второстепенное действие; destructive (обводка/текст `color.error`) — «Отклонить», «Удалить» |
| Состояния | default, hover (затемнение на 8%), active, disabled (opacity 40%, курсор not-allowed), loading (спиннер вместо/рядом с текстом) |
| Доступность | видимый focus-индикатор (обводка 2px `color.primary`), доступна с клавиатуры (Enter/Space), `aria-disabled` в состоянии disabled |

### Компонент: «Поле ввода / текстовая область»

| Параметр | Описание |
|---|---|
| Назначение | Ввод данных в формах: подача статьи, рецензия, решение, регистрация |
| Структура | лейбл над полем, само поле, опциональная подсказка снизу, сообщение об ошибке снизу (заменяет подсказку) |
| Размеры | height 40px для однострочных полей, textarea — минимум 96px с возможностью растягивания (для аннотаций, комментариев) |
| Варианты | текстовое поле, textarea, select (выпадающий список — например, выбор рекомендации рецензента), file upload (загрузка рукописи/документов/рецензии) |
| Состояния | default, focus (обводка `color.primary`), error (обводка `color.error` + текст ошибки под полем), disabled, filled |
| Доступность | `<label for>` жёстко связан с полем, ошибка помечена `aria-describedby` и `aria-invalid`, обязательные поля помечены визуально и текстом «обязательное поле», а не только звёздочкой |

### Компонент: «Форма подачи/рецензирования» (композитный)

| Параметр | Описание |
|---|---|
| Назначение | Многошаговая форма для подачи статьи (US-1, US-3, US-8), заполнения рецензии (US-6) и вынесения редакционного решения (US-4, US-7) — единый паттерн композиции полей ввода |
| Структура | заголовок шага (`type.h3`), группы полей ввода, индикатор шагов (для подачи статьи — «Метаданные → Файлы → Проверка»), кнопки «Назад»/«Далее»/«Отправить» |
| Размеры | занимает правую колонку личного кабинета (10/12), поля — в одну или две колонки в зависимости от типа |
| Варианты | «форма подачи статьи» (несколько шагов), «форма рецензии» (один экран: анкета + рекомендация + файл), «форма решения» (один экран: комментарий + выбор решения) |
| Состояния | пустая форма, частично заполненная (черновик — для формы подачи, автосохранение), ошибка валидации (см. компонент «Поле ввода»), отправка (кнопка в состоянии loading) |
| Доступность | последовательный порядок табуляции по полям, `aria-live` для сообщения об успешной отправке, предупреждение при попытке уйти со страницы с несохранёнными данными |

| Размеры | компактный блок/модальное окно, вызывается по кнопке |
| Варианты | инлайн-блок на странице статьи либо модальное окно (уточняется на этапе вёрстки) |
| Состояния | обычное; после копирования — временное сообщение «Скопировано» |
| Доступность | результат копирования объявляется через `aria-live="polite"` |

### Компонент: «Уведомление» (Toast и список в личном кабинете)

| Параметр | Описание |
|---|---|
| Назначение | Toast — мгновенная обратная связь по действию (см. TS раздел 10); список уведомлений — раздел «Уведомления» в личном кабинете |
| Структура | Toast: иконка типа (успех/ошибка/инфо), текст сообщения, автозакрытие; элемент списка: иконка типа события, текст, дата, статус прочитанности (точка-индикатор) |
| Размеры | Toast — фиксированная позиция (например, верхний правый угол), max-width 360px; элемент списка — во всю ширину контентной области |
| Варианты | success, error, warning, info — для Toast; типы событий (статус изменён, приглашение, комментарий, решение) — для списка |
| Состояния | Toast: появление/исчезновение (см. раздел 9 Motion); элемент списка: прочитано/непрочитано (непрочитанное — фон `color.surface` и жирный текст) |
| Доступность | Toast объявляется через `aria-live="assertive"` для ошибок и `"polite"` для успеха; непрочитанные уведомления доступны через счётчик с `aria-label` («5 непрочитанных уведомлений») |

### Компонент: «Модальное окно»

| Параметр | Описание |
|---|---|
| Назначение | Подтверждение критичных действий (например, «Отклонить статью», выход из аккаунта), блок цитирования |

| Размеры | max-width 480px, центрируется по экрану, затемнение фона (overlay) |
| Варианты | подтверждающее (Ок/Отмена), информационное (только Ок) |
| Состояния | открытие/закрытие (`shadow.lg`, transition из раздела 9) |
| Доступность | focus trap внутри модалки, закрытие по Esc, возврат фокуса на элемент, открывший модалку, `role="dialog"` и `aria-modal="true"` |

## 8. Accessibility

- Соответствие: WCAG 2.1 **AA**
- Контраст текста: минимум 4.5:1 (обычный текст), 3:1 (крупный текст от 24px) — все токены `color.text.*` и `color.status.*` из раздела 2 подобраны с учётом этого требования к фону `color.background`/`color.surface`
- Управление с клавиатуры: все интерактивные элементы (кнопки, поля, карточки-ссылки, PDF-viewer, вкладки статусов) доступны через Tab, видимый focus-индикатор (обводка 2px `color.primary`)
- ARIA: модальные окна — `role="dialog"`, `aria-modal="true"`, focus trap; Toast-уведомления — `aria-live`; статус-бейджи и индикаторы срока (приглашения) — статус передаётся текстом, не только цветом (принцип 5 раздела 1)
- Альтернативный текст для изображений: обязателен для обложек выпусков, иллюстраций пустых состояний; декоративные иконки помечаются `aria-hidden="true"`
- Формы: жёсткая связка `label` ↔ `input` (`for`/`id`), понятные текстовые сообщения об ошибках под полем, ошибки также объявляются через `aria-live` при отправке формы
- PDF-viewer: доступна прямая ссылка на скачивание файла как альтернатива встроенному просмотрщику для пользователей ассистивных технологий

## 9. Motion

| Переход | Длительность | Easing | Когда применяется |
|---|---|---|---|
| Наведение (hover) | 150ms | ease-out | кнопки, карточки |
| Появление модалки | 250ms | ease-in-out | модальные окна |
| Появление/скрытие Toast | 200ms | ease-out | всплывающие уведомления |
| Смена страницы | 200ms (fade) | ease-in-out | переход между публичными страницами (SSR/пререндер); в личном кабинете (SPA) переходы между разделами — без анимации или мгновенный fade 100ms, чтобы не замедлять частую навигацию по спискам статей |

*Общее правило: анимация не должна мешать пользователям с вестибулярными нарушениями — предусмотрен `prefers-reduced-motion`, при котором все переходы заменяются мгновенной сменой состояния без движения.*
