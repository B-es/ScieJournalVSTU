# Процесс работы научного журнала

## Роли и обязанности

| Роль | Обязанности |
|------|-------------|
| **Автор** | Регистрация, подача рукописи + метаданные, получение уведомлений, загрузка исправленной версии. |
| **Главный редактор** | Проверка соответствия тематике, назначение одного рецензента, принятие решения (принять / доработать / отклонить) на основе рецензии и после доработки. |
| **Рецензент** | Получение приглашения, принятие/отказ, загрузка рецензии с рекомендацией и комментариями. |
| **Технический редактор (админ)** | Проверка оформления, присвоение DOI, вёрстка, публикация в выпуске и архиве, управление пользователями и настройками через админку. |

---

## Пошаговый процесс

### 1. Подача статьи (Автор)
- Заполняет форму: название (рус/англ), аннотация, ключевые слова, список авторов, файл рукописи.
- Система присваивает статус *«На рассмотрении»*, уведомляет Главного редактора.

### 2. Предварительная проверка (Главный редактор)
- Просматривает рукопись на соответствие тематике и минимальным требованиям.
- **Если нет** → отклоняет с комментарием, статус *«Отклонена»*, процесс завершён.
- **Если да** → переходит к назначению рецензента.

### 3. Назначение рецензента (Главный редактор)
- Выбирает одного рецензента из базы (или вводит email внешнего).
- Система отправляет приглашение с аннотацией и сроком.

### 4. Принятие приглашения (Рецензент)
- Рецензент входит в систему, нажимает «Принять» или «Отказать».
- **Отказ** → уведомление Главному редактору, возврат к шагу 3.
- **Принято** → статус *«Рецензируется»*, доступ к полному тексту.

### 5. Рецензирование (Рецензент)
- Заполняет форму: оценки по критериям, комментарии, итоговая рекомендация (принять / доработать / отклонить).
- Отправляет рецензию → уведомление Главному редактору.

### 6. Принятие решения (Главный редактор)
- Изучает рецензию, принимает одно из решений:
  - **Принять** → статус *«Принята к публикации»*, передача Техническому редактору (шаг 8).
  - **Отклонить** → статус *«Отклонена»*, автор получает мотивированный отказ, процесс завершён.
  - **На доработку** → статус *«Требуется доработка»*, автор получает комментарии рецензента → шаг 7.

### 7. Доработка (Автор)
- Загружает исправленную версию рукописи (и опционально ответы на замечания).
- Система уведомляет Главного редактора.
- **Повторное принятие решения**: Главный редактор анализирует исправленную версию и снова принимает решение (шаг 6):
  - **Принять** → шаг 8.
  - **Отклонить** → статус *«Отклонена»*, процесс завершён.

### 8. Техническая подготовка и публикация (Технический редактор)
- Проверяет оформление и метаданные.
- Присваивает DOI.
- Выполняет минимальную вёрстку (формирует финальный PDF).
- Публикует статью в выпуске и архиве → статус *«Опубликована»*.
- Рассылает уведомления автору и заинтересованным лицам.
- Процесс завершён.

## Объекты данных (артефакты)

| Объект | Создаётся | Используется |
|--------|-----------|--------------|
| Рукопись (первая версия) | Шаг 1 | Шаги 2, 5, 6 |
| Рецензия | Шаг 5 | Шаг 6 |
| Исправленная рукопись | Шаг 7 | Шаг 6 (повторно) |
| DOI | Шаг 8 | Хранится со статьёй |
| Финальный PDF | Шаг 8 | Публикуется и архивируется |
| Архив журнала | Шаг 8 | Хранилище опубликованных статей |

---

## Статусы статьи

| Статус | Когда |
|--------|-------|
| На рассмотрении | После подачи (шаг 1) |
| Рецензируется | После принятия приглашения рецензентом (шаг 4) |
| Требуется доработка | После решения «на доработку» (шаг 6) |
| Принята к публикации | После положительного решения (шаг 6 или повторное) |
| Отклонена | После отклонения (шаг 2 или шаг 6) |
| Опубликована | После шага 8 |

---

## Уведомления (email + личный кабинет)

| Событие | Получатель |
|---------|------------|
| Подана новая статья | Главный редактор |
| Отклонение на предпроверке | Автор |
| Приглашение рецензенту | Рецензент |
| Отказ рецензента | Главный редактор |
| Рецензия загружена | Главный редактор |
| Решение «на доработку» | Автор |
| Решение «отклонить» | Автор |
| Решение «принять» | Автор, Технический редактор |
| Статья опубликована | Автор, соавторы |

---

## Административная панель (Технический редактор)

- Управление пользователями (роли, блокировка).
- Настройка журнала (разделы, сроки, шаблоны).
- Управление выпусками.
- Публикация статей (шаг 8).
- Просмотр статистики.

---

## Мультиязычность (русский / английский)

- Интерфейс, формы, уведомления и письма доступны на двух языках (выбор пользователя).
- Метаданные статьи (название, аннотация) запрашиваются на обоих языках.

---

## Ключевые принципы (упрощения)

- Один рецензент на статью.
- После доработки решение принимает Главный редактор (без повторного рецензирования).
- Сроки контролируются вручную через уведомления (без автоматических таймеров).
- Вёрстка минимальная (автоматическая генерация PDF).
